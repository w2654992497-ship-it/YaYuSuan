# 鸭鱼算 · 产品需求文档（PRD）

> 版本：v1.0 · 日期：2026-03-31

---

## 一、产品概述

### 1.1 产品目标

鸭鱼算是一款运行在 HarmonyOS 上的轻量个人账单管理工具。用户可以快速记录每笔收支，通过图表直观了解自己的消费结构，并可选择设置预算进行支出管控。

### 1.2 目标用户

有记账习惯或想建立记账习惯的个人用户，无需财务背景知识，操作流程极简。

### 1.3 核心功能

| 优先级 | 功能 |
|--------|------|
| P0（必须） | 账单录入、账单列表、统计图表 |
| P1（重要） | 条件筛选、滑动删除、月份切换 |
| P2（可选） | 预算提醒、数据导入导出 |

---

## 二、信息架构

```
鸭鱼算
├── 账单（BillList）          ← 默认启动 Tab
│   ├── 收支汇总卡片
│   ├── 筛选栏
│   └── 账单列表（按日期分组）
│       └── 单条账单（右滑删除）
├── 记账（AddBill）
│   ├── 收入/支出切换
│   ├── 金额输入
│   ├── 分类选择
│   ├── 日期选择
│   └── 备注输入
└── 统计（Statistics）
    ├── 月份导航
    ├── 月度汇总
    ├── 饼图（分类占比）
    └── 柱状图（近6个月趋势）

[可选] 设置（Settings）         ← 从账单页右上角图标进入
    └── 各分类预算设置
```

## 三、全局交互规范

| 规则 | 说明 |
|------|------|
| 底部导航 | 固定 3 个 Tab：账单 / 记账 / 统计，图标 + 文字，选中时主题色高亮 |
| 加载状态 | 数据库查询期间显示 `LoadingProgress`，避免白屏 |
| 空状态 | 列表无数据时显示插图 + "暂无账单，快去记一笔吧" |
| 错误提示 | 数据库操作失败时 `showToast`，不崩溃 |
| 金额显示 | 收入前缀 `+`，绿色；支出前缀 `-`，红色；保留2位小数 |
| 日期显示 | 今天显示"今天"，昨天显示"昨天"，其余显示"M月D日 星期X" |

### 4.1 Index · 主容器页

#### 职责

作为所有页面的宿主容器，提供底部 Tab 导航，管理页面间状态同步。

#### 布局设计

```

│   [当前 Tab 内容区域]        │
│                             │
│                             │
│                             │
│                             │
├─────────────────────────────┤
│  [账单]    [记账]    [统计]  │  ← TabBar，高度 56vp
└─────────────────────────────┘
```

#### 交互需求

| 编号 | 触发 | 行为 |
|------|------|------|
| I-01 | 点击"账单" Tab | 切换到 BillList，Tab 图标高亮 |
| I-02 | 点击"记账" Tab | 切换到 AddBill，Tab 图标高亮 |
| I-03 | 点击"统计" Tab | 切换到 Statistics，Tab 图标高亮 |
| I-04 | AddBill 提交成功 | 自动切换到"账单" Tab 并刷新列表 |

#### 状态说明

- `currentIndex` 默认为 `0`（账单页）
- AddBill 提交成功后通过 `AppStorage.set('needRefresh', true)` 通知 BillList

- 备注为空：BillItem 中间列只显示分类名，不显示第二行

#### 金额输入区交互

```
点击金额区域
    ↓

    ↓
页面整体上移（键盘避让）
    ↓
金额实时更新显示
    ↓
点击其他区域 / 收起键盘
    ↓
金额格式化（保留2位小数）
```

#### 交互需求

| 编号 | 触发 | 行为 |
|------|------|------|
| AB-01 | 页面进入 | 默认选中"支出"，金额清空，日期为今天，分类未选中 |
| AB-02 | 点击"收入"/"支出"切换 | 分类宫格切换对应类型的分类列表，已选分类清空 |
| AB-03 | 点击金额区域 | 弹出数字键盘，光标置于末尾 |
| AB-04 | 输入金额 | 实时展示，限制：最多8位整数 + 2位小数，不允许负数 |
| AB-05 | 输入 `0` 开头 | 自动格式化为 `0.xx`，防止 `00123` |
| AB-06 | 点击分类格 | 该格高亮，其余恢复，`selectedCategory` 更新 |
| AB-07 | 点击日期行 | 弹出 `DatePickerDialog`，默认当前已选日期，不可选未来日期 |
| AB-08 | 选择日期确认 | 日期行文字更新，Dialog 关闭 |
| AB-09 | 点击备注输入 | 键盘弹出，maxLength 50，超出不可继续输入 |
| AB-10 | 点击"保存" | 触发表单验证（见下方） |
| AB-11 | 验证失败-金额为空 | 金额区域轻微抖动动画 + Toast："请输入金额" |
| AB-12 | 验证失败-金额为0 | Toast："金额必须大于0" |
| AB-13 | 验证失败-未选分类 | CategoryGrid 标题变红 + Toast："请选择分类" |
| AB-14 | 验证通过 | 调用 `BillDao.insert()`，成功后清空表单，切换到账单 Tab |
| AB-15 | 数据库写入失败 | Toast："保存失败，请重试" |

#### 表单验证逻辑

```
点击保存
  ├── 金额为空？        → 报错 AB-11
  ├── 金额 <= 0？       → 报错 AB-12
  ├── 未选分类？        → 报错 AB-13
  └── 全部通过          → 构造 Bill 对象 → insert → 跳转
```

#### 边界情况

- 金额输入时连续点小数点：忽略第二个小数点
- 金额全部删除后显示占位 `¥ 0.00`
- 备注粘贴超过 50 字：自动截断到 50 字

#### 交互需求

| 编号 | 触发 | 行为 |
|------|------|------|
| ST-01 | 页面进入 | 默认显示当月，加载饼图数据（支出分类统计） |
| ST-02 | 点击"←"箭头 | 月份减一，重新查询并刷新汇总卡片和图表 |
| ST-03 | 点击"→"箭头 | 月份加一，不可超过当月（当月时箭头禁用置灰） |
| ST-04 | 点击"趋势柱图" Tab | 切换到柱状图，显示含当前月在内的近6个月数据 |

- 近6个月某月无数据：柱高度为0，X 轴标签正常显示
- 收入分类占比统计：饼图默认显示支出，暂不支持切换收入分布（v1.0）

### 4.5 Settings · 预算设置页（可选 P2）

#### 职责

用户为每个支出分类设置月度预算，超出预算时在录入时给予提醒。

#### 布局设计

```
┌─────────────────────────────┐
│ ← 预算设置                   │  ← 顶部导航栏（从账单页右上角进入）
│ ─────────────────────────── │
│                             │
│  🍜 餐饮          [¥  500] │  ← 每行：图标+名称 + 输入框
│  🚇 交通          [¥  300] │
│  🛍 购物          [¥  800] │
│  🎮 娱乐          [¥  200] │
│  🏠 住房          [¥ 2000] │
│  💊 医疗          [¥  500] │
│  📚 教育          [¥  300] │
│  ··· 其他         [¥  200] │
│                             │
│   输入 0 表示不限制该分类    │  ← 说明文字，灰色
└─────────────────────────────┘
```

#### 交互需求

| 编号 | 触发 | 行为 |
|------|------|------|
| SE-01 | 页面进入 | 从 Preferences 读取各分类预算，填入输入框；无则显示 `0` |
| SE-02 | 修改输入框 | 实时保存到 Preferences，无需确认按钮 |

| SE-04 | 设置为 `0` | 表示该分类不限制预算 |

#### 预算超额提醒逻辑（在 AddBill 保存时触发）

```
用户点击保存（type = expense）
    ↓
读取该分类预算 budget（来自 Preferences）
    ↓
budget = 0？→ 不提醒，直接保存
    ↓
查询本月该分类已支出总额 used
    ↓
used + 本次金额 > budget？
    ↓
弹出 Dialog：
"⚠️ 餐饮预算超额
本月预算 ¥500，已支出 ¥480，
本次记录后将超出 ¥5。
[取消]  [仍然保存]"
```

## 七、异常情况处理汇总

| 场景 | 处理方式 |
|------|---------|
| 数据库初始化失败 | `EntryAbility` 中 `showDialog` 提示"初始化失败，请重启应用" |
| 查询返回空数组 | 显示空状态视图，不报错 |
| 插入/删除失败 | `showToast("操作失败，请重试")` |
| 金额输入溢出（超过8位） | 输入框不接收后续字符 |
| Canvas 容器尺寸为0 | `onReady` 中判断 `width === 0` 则不绘制 |
| 导入文件格式错误（可选） | `showDialog` 提示"文件格式不支持，请选择 JSON 或 TXT 文件" |
| 导入数据重复（可选） | 按 `createdAt` 去重，跳过已存在记录，提示"已跳过 X 条重复记录" |
