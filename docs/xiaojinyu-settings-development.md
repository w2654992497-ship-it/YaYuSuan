# 小金鱼记账设置页开发说明

> 适用版本：1.0.0  
> 正式应用名：小金鱼记账  
> 对应页面：`entry/src/main/ets/pages/Settings.ets`

## 1. 设置页合理性分析

新设置页从“单一预算设置页”升级为“账户、数据、预算、通用设置”的综合设置中心是合理的。

原因如下：

- 记账产品的设置页不应只承载预算。用户会自然寻找数据备份、隐私、安全、导出、分类管理、关于版本等入口。
- 当前应用已经有分类管理、预算保存、CSV 导出能力，把它们收敛到设置页，比散落在账单页或隐藏按钮中更容易发现。
- “账户与数据”放在首屏上方合理，因为数据安全是记账产品的核心信任点。
- “记账与预算”独立成组合理，因为分类、预算、周期账单属于高频维护项。
- “通用设置”承载主题、安全、反馈、关于，符合常见 App 信息架构。
- 底部保留“退出登录”视觉入口合理，但当前版本如果没有真实账号体系，应先作为占位提示，避免误导用户以为已接入账号服务。

## 2. 是否需要添加新功能

需要添加，但必须分层落地，避免一次性接入过多高风险能力。

当前版本建议落地：

- 分类管理：已存在，作为真实功能入口。
- 预算设置：已存在，改为设置页内展开面板，避免跳转层级过深。
- CSV 导出：已存在，保留为数据迁移能力。
- 清除缓存：轻量功能，当前可先做 Toast 反馈，后续接真实缓存清理。
- 关于小金鱼记账：显示版本信息。

后续版本建议接入：

- 数据备份：需要明确本地备份、云备份或系统 BackupExtension 的策略。
- 多设备同步：需要账号体系和冲突合并策略，不建议仅做 UI。
- 隐私设置：建议包含隐私政策、数据权限说明、个性化开关。
- 安全与密码：建议接入应用锁、数字密码或生物识别能力。
- 周期记账：需要新增周期规则模型、执行策略和下次生成时间。
- 意见反馈：需要接入邮箱、网页表单或服务端接口。
- 主题模式：需要全局主题状态和深色模式资源联动。
